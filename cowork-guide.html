Is Cowork the right Claude for you?

Anthropic ships three different ways to work with Claude, and they're not interchangeable. If you read this whole guide and then realize you actually wanted Claude.ai or Claude Code, that's an hour you can't get back. Quick disambiguation first.

Web & mobile

Claude.ai

Pick this if you want pure conversation — chat with Claude, paste in text or images, get answers. No files, no app installs, no real-world action.

Browser-based, also iOS and Android apps Best for questions, drafting, explaining Can use connectors and artifacts Won't touch your local files or apps
You are here

Claude Cowork

Pick this if you want Claude alongside you on your real Mac — reading and writing your files, driving your apps, remembering your projects, running on a schedule.

Native macOS desktop app Best for work that touches your files and tools Has memory, scheduled tasks, computer use, MCP You give it folder access and connector permissions per use
Terminal

Claude Code

Pick this if you write software — Claude as an agentic coding partner inside your terminal, working in your repo, running tests, opening PRs.

CLI tool, runs in your terminal Best for code, tests, refactors, deploys Knows git, runs commands, edits files Built for engineers, not general users
Quick decision tree

Do you write code for a living? → Claude Code probably fits better, though Cowork can complement it. Are you trying to get conversational answers without touching files? → Claude.ai is lighter. Are you trying to make Claude do real work across your apps, files, and calendar — like a coworker would? → You're in the right place. Keep reading.

Day zero

Install Cowork.

Five steps from "I don't have it" to "I'm running my first task." Maybe 10 minutes if your downloads are fast and you're already signed into Claude.

1

Check that Cowork is available to you

Cowork is currently a macOS desktop application from Anthropic, distributed via the Claude.ai download page. Availability and plan requirements may vary by region and over time — before downloading, confirm at claude.com or your account dashboard that Cowork is available on your plan.

Heads up: Cowork is a desktop app, not a browser feature. If you're on a Chromebook, iPad, or Linux, the desktop app won't run. Use Claude.ai on the web instead.
2

Download and install the desktop app

From the Claude downloads page, grab the macOS installer. It'll be a .dmg file you double-click. Drag the Claude app icon into your Applications folder when prompted.

On first launch, macOS may ask whether you really want to open an app downloaded from the internet. Confirm yes. The app icon will appear in your Dock and menu bar.

3

Sign in to your Claude account

Use the same login you'd use at claude.ai. The desktop app uses your Claude account for sync, plan recognition, and connector OAuth. If your plan doesn't include Cowork access, you'll see Chat mode but not Cowork mode. Upgrade or switch plans from your account settings.

4

Switch to Cowork mode

Once signed in, the app opens to Chat mode by default. Look at the top of the conversation pane: there's a tab switcher with Chat, Cowork, and Code. Click Cowork.

The first time you switch to Cowork, you'll see a brief intro and a "Let's knock something off your list" landing page. From here you can start a new task, open Projects, see Scheduled tasks, browse Live artifacts, or open the Customize panel.

5

Pick a working folder and try a task

Before doing real work, give Cowork a folder to read and write in. Make a folder called something obvious — ~/Documents/CoworkDemo works fine for a first test — and select it when Cowork asks.

Then run a real task. Don't test with a toy prompt — ask Cowork to do something you'd otherwise do yourself this morning. The Quick Start section walks through three example first tasks.

Connectors and permissions come later. You don't need to set up Gmail, Slack, or any other connector to start. Cowork works with just a folder. Add connectors as you discover you want them — each one is a separate OAuth approval, takes about a minute, and can be revoked anytime. The Trust & Data section covers the security model in detail.

Quick start

Get to your first win in five minutes.

If you do nothing else from this guide, do these three things. After that, Cowork can already do useful work.

1

Pick a folder for Claude to work in

Cowork needs a workspace. Pick a folder where finished outputs should land — a project folder, a Documents subfolder, anything sensible. Claude can read and write files there. Files outside that folder are off-limits unless you grant access.

Try saying
"Let's use my ~/Documents/Projects/marketing folder for this session."
Heads up: Cowork has two file areas. The folder you pick (where your real work lives) and a hidden temporary scratchpad (where Claude drafts things you don't need to see). Final outputs always land in your folder with a clickable link.
2

Connect one tool you use every day

A connected tool turns Claude from a smart chat into a coworker. Start with the one you live in. Gmail, Calendar, Slack, Notion, or Linear are common first picks.

Try saying
"Connect Gmail so you can help me triage email."
How it works: Claude opens an OAuth flow in your browser. You approve, and the connection persists across sessions. You can revoke it anytime.
3

Run one real task end to end

Don't test Cowork with a toy prompt. Ask it to do something you'd otherwise do yourself today. The point is to see it complete a real loop — read source data, do the work, hand you a deliverable.

Permissions & data

What Cowork can — and can't — do with your stuff.

If you're going to give an AI access to your email, calendar, files, and screen, you should know exactly what it sees and what it doesn't. This is the trust model in plain English. For the legal version, check Anthropic's privacy policy and DPA.

Folder access is bounded

When you select a folder for Cowork to work in, it can read and write files in that folder only. Files elsewhere on your Mac — Desktop, other Documents folders, system files — are off-limits unless you explicitly grant access to them too.

Connectors use OAuth, not passwords

When you connect Gmail, Slack, Notion, etc., you approve access through that service's standard OAuth flow. Cowork never sees your password — just a scoped token from the service. You can revoke any token from your account dashboard for that service at any time.

Per-app desktop permissions

Computer use (driving native apps via clicks/screenshots) requires you to explicitly approve each app you let Cowork control. Approvals are scoped per session by default and shown in a dialog you click. You can deny any app and Cowork respects it.

Memory is local-first

Cowork's persistent memory (what it remembers about you across sessions) lives as files on your Mac, inside Anthropic's app data folder. You can read, edit, or delete those files directly. Uninstalling the app removes the local memory.

Conversations go to Anthropic's servers

Like Claude.ai, your prompts and Cowork's responses are processed by Anthropic's models. That includes file contents Cowork reads on your behalf when responding to a prompt. Don't share content with Cowork that you wouldn't share with a third-party AI provider — the same way you'd treat any cloud tool.

Training-on-data depends on your plan

Whether your inputs and outputs are used to improve Claude's models depends on your plan and account settings. Pro, Team, and Enterprise plans generally have stricter defaults than Free. Check your account's data settings to confirm the current behavior, and review Anthropic's privacy policy for the authoritative answer.

Some actions are off-limits even if you ask

By design, Cowork won't execute trades, send money, initiate transfers, modify security permissions, or delete things in ways that can't be undone — even when explicitly asked. It'll route you to do those yourself. This is intentional safety, not a bug.

Verify before destructive operations

Bulk renames, batch deletes, mass email sends, file moves — Cowork will do these, but you should ask for a dry run first ("show me what you'd do before doing it"). That's not Cowork being cautious; it's you protecting yourself from a misunderstood prompt.

For specifics: Anthropic's privacy policy and acceptable use policy are the authoritative documents. This section is plain-English orientation, not legal text. If you're handling regulated data (HIPAA, FERPA, GDPR-covered, etc.), check with your organization's compliance team before connecting work accounts.

Worked example

Activate a role-bundle plugin.

Five steps, real screenshots. We're using the Design plugin in this walkthrough — the exact same flow works for Sales, Marketing, Engineering, Legal, Ops, Product Management, Customer Support, or Brand Voice. Pick whichever matches your role.

1

Open Customize

From any Cowork window, click Customize in the left sidebar (briefcase icon, near the bottom). You'll land on the page below.

The "Customize Claude" landing page. Three cards correspond to the three ways to extend Cowork — connectors, skills, and plugins.

2

Click "Browse plugins"

The Directory opens to the Plugins tab, showing role bundles from Anthropic & Partners. Each card lists the publisher, install count, and a one-line description of what the bundle covers.

Ten role bundles available right now. Each one packages 5–10 skills plus recommended connectors for that job function.

3

Open the plugin you want

Click any plugin card to see what's inside. The detail view shows:

  • ·
    "Try asking…" examples — sample prompts that fire each skill, so you can see what the bundle actually does.
  • ·
    Skills chips — the slash-command names you can use to trigger each skill directly.
  • ·
    Connectors chips — tools the plugin works best with. Not required, but the skills get richer when these are connected.

The Design plugin: 5 example prompts, 7 skills (/design-critique, /accessibility-review, and 5 more), and 9 recommended connectors (Slack, Figma, Linear, Asana, Atlassian, Notion, Intercom, Google Calendar, +1).

4

Hit Install

Top-right of the plugin detail view: an Install button. Click it. The plugin downloads and registers all its skills in your Cowork instance.

If you've already installed the plugin, the button reads Uninstall and a second Manage button appears next to it. Manage lets you turn individual skills on/off without uninstalling the whole bundle — useful when you want, say, 5 of the 7 Design skills but not all.

Connectors aren't auto-installed. To finish the setup, switch to the Connectors tab in the Directory and connect the tools the plugin recommends. OAuth flow per service, one minute each. Skip any you don't use.

5

Use it — two ways

Once installed, the plugin shows up in your left sidebar under Personal plugins, with every skill nested below it. Skills activate in two ways:

Natural language (default). Just describe what you need. Each skill has a description telling Cowork when to fire. "Critique this design for me — I'm worried about hierarchy" auto-activates /design-critique without you naming it.
Slash command (explicit). Type the skill's name with a leading slash to force it. /design-handoff jumps straight into the developer handoff workflow even if your prompt was ambiguous about which skill to use. Best for when you know exactly what you want.

That's it. The plugin is live. Skills will trigger on their own from now on whenever your prompts match their patterns — no further setup needed.

One more thing — making your own. The plugin marketplace is the easy path, but you can also build custom skills. From Customize → Skills, click + and pick Create with Claude (conversational builder), Write skill instructions (a form for the name, description, and instructions), or Upload a skill (drop in a .skill zip). Custom skills activate the same way as bundled ones — natural language match or slash command.

Generated PRD — opening section

Product Requirements: Cross-platform social composer

Problem

Users scheduling posts across LinkedIn and Twitter currently maintain two parallel workflows. Each platform has different character limits, link preview behavior, and image requirements. The result: same content gets rewritten twice, tone drifts between platforms, and posts get scheduled at non-optimal times because the user is tired of the second pass.

Goals

  • One composer that handles both platforms with per-platform adjustments
  • Preview shows what each version will look like before scheduling
  • Optimal-time suggestions based on the user's engagement history

Non-goals

  • Direct message scheduling (separate feature)
  • Reply / comment management (out of scope)

When things go sideways

Common pitfalls — with the actual fix.

Things break, configurations get weird, prompts don't trigger the skill you expected. The catalog below covers the most common failure modes a new user hits in their first month. Each entry: symptom, root cause, fix.

Symptom

"Cowork can't see my emails."

Likely cause: The Gmail OAuth token expired (this happens periodically by design) or the original connection scope didn't include the mailbox you're asking about.

Fix: Open Customize → Connectors → Gmail. Disconnect and reconnect. When the OAuth dialog appears, make sure you're approving for the right Google account — if you have multiple, the picker defaults to your last-used.
Symptom

"My scheduled task didn't fire."

Likely cause: Your laptop was asleep or shut down at the scheduled time. Cowork's scheduled tasks run while the computer is awake — they don't fire from the cloud.

Fix: In Cowork → Scheduled, toggle Keep awake on. This prevents sleep when a task is approaching. Plug in if the task fires overnight. For tasks that really need to fire when you're away, the cloud-scheduled approach lives on the roadmap; for now, dedicate a machine that stays on.
Symptom

"Cowork wrote a file but I can't find it."

Likely cause: Cowork has two file areas — your selected working folder (visible to you) and a hidden scratchpad (where it drafts things you don't need to see). It dropped the file in the scratchpad, not your folder.

Fix: Tell it explicitly where to put outputs. "Save the final report to my working folder" or "put it in this folder" usually does the trick. Final deliverables should always come back with a clickable link in chat — if there's no link, it didn't land in your folder.
Symptom

"The skill I expected didn't activate."

Likely cause: Your prompt didn't match the skill's description closely enough, so Cowork didn't auto-trigger it. Skills fire based on description match, not feature lists.

Fix: Use the slash command explicitly. /design-critique forces the design critique skill regardless of phrasing. Or rephrase your prompt to include the verbs the skill expects ("critique," "review," "audit," etc.).
Symptom

"Cowork refused to do something on my desktop."

Likely cause: The app you asked it to drive isn't in the current session's permission allowlist. Computer-use access is per-app, per-session, and Cowork won't act outside what you've approved.

Fix: Cowork will request access for the app it needs. Approve the dialog when it appears. If the app is in a restricted tier (browsers and IDEs default to read-only for safety), use the Claude in Chrome extension or run shell commands instead, depending on the task.
Symptom

"My memory seems wrong or outdated."

Likely cause: A memory file captured a fact when it was true and Cowork hasn't realized it's changed. Memory doesn't auto-expire.

Fix: Just tell Cowork: "Update your memory — my role is now X, not Y." Or audit memory directly: ask "show me your memory files" and edit or delete entries that are stale. The files live as plain text on disk; you can edit them in any text editor too.
Symptom

"A connector keeps logging me out."

Likely cause: Some services (Microsoft 365 in particular) have shorter token lifetimes for security reasons, so connections expire and need refreshing.

Fix: Reconnect when prompted. If it's happening often, check whether your organization's IT has set tighter token policies — that overrides the default lifetime.
Symptom

"I gave Cowork a big task and it stopped halfway."

Likely cause: You hit a usage limit (token quota for your plan) or the conversation got too long and ran out of context. Both are common on long agentic tasks.

Fix: For limits, wait til reset (you'll see a message with the reset time) or upgrade your plan. For long tasks, use Dispatch instead of a regular conversation — it handles long-running work better. For huge research tasks, break into stages: have Cowork hand off intermediate state as files between stages.

Glossary.

Every term Cowork uses, in plain English. Bookmark or print this if you're new — the vocabulary moves fast in the rest of the guide.

Cowork
The mode of the Claude desktop app that gives Claude access to your files, computer, and connected tools. Distinct from Chat mode (conversation-only) and Code mode (Claude Code).
Project
A folder + a set of instructions Cowork associates with ongoing work. Memory and context build up over time. Equivalent of a workspace.
Working folder
The specific folder on your Mac Cowork can read and write in for a given conversation. Files outside it are off-limits unless you grant separate access.
Skill
A recipe Claude follows to do a specific kind of task — writing a PRD, critiquing a design, triaging an NDA. Skills activate on natural-language match or slash command.
Slash command
An explicit way to invoke a skill by typing /skill-name. Use when you want to force a specific skill regardless of how your prompt is phrased.
Plugin
A bundle of related skills plus recommended connectors, usually role-themed (Design, Sales, Engineering). Installing a plugin activates all its skills at once.
Connector
An integration with an external service (Gmail, Slack, Notion, etc.) that gives Cowork read/write access through that service's API. Authorized via OAuth.
MCP
Model Context Protocol — the open standard Anthropic uses for connectors. Means you can also build or install community-made integrations for tools Anthropic doesn't ship official connectors for.
Memory
Persistent notes Cowork keeps about you across sessions: your role, preferences, feedback you've given, references to external systems. Lives as text files on your Mac.
Scheduled task
A task that fires on a recurring cadence (every weekday at 7 AM) or at a future time (in an hour). Runs while your computer is awake.
Dispatch
A long-running background task Cowork works on asynchronously. You give it a goal, walk away, check in later. Beta feature.
Live artifact
A self-contained HTML page Cowork builds that persists and refreshes its data each time you open it. Good for trackers, dashboards, recurring reports.
Subagent
A specialized helper Cowork can dispatch in parallel for heavy work (deep research, cross-codebase search, independent code review). You usually won't summon them directly — Cowork picks them automatically.
Computer use
Cowork's ability to take screenshots of your screen and drive native apps with mouse and keyboard. Permission-gated per app.
Customize panel
The configuration area in Cowork (briefcase icon, left sidebar) where you manage Skills, Connectors, and Plugins.
Directory
The marketplace inside Cowork where you browse and install skills, connectors, and plugins published by Anthropic and partners.
Artifact (Chat)
Distinct from "Live artifact." In Chat mode, an artifact is just a generated document or piece of code shown alongside the conversation, with no live data refresh.
OAuth
The standard way connectors authenticate. You approve access through the service's own login page; Cowork gets a scoped token, never your password. Revocable anytime.
Tips & patterns

Get more out of Cowork.

Specificity

Tell it what "done" looks like.

"Summarize this" leaves Claude guessing about length, format, and audience. "Summarize this for a CFO in 5 bullets, under 100 words" gets you the right answer first try.

Iteration

Treat the first draft as a starting point.

Cowork is fast at revision. Don't try to nail it in one prompt — get a v1, then sharpen with "make it more direct", "cut the intro", "add a chart".

Context

Connect more, prompt less.

Every connector you add reduces what you need to type. Once Gmail and Calendar are connected, "prep me for tomorrow's meetings" becomes a one-liner.

Memory

Train it on your preferences.

When Claude does something well, say so. When it overdoes it, correct it. Both go into memory. Within a week it'll feel personalized.

Safety

Verify before destructive actions.

Before letting Claude bulk-rename, bulk-delete, or send emails to multiple people, ask for a dry run first: "show me what you'll do before doing it."

Delegate

Push hard work to scheduled tasks.

If you find yourself asking the same question every morning, every Monday, or every month — that's a scheduled task waiting to happen.

Cowork Cheat Sheet

v1 · claude-cowork

Sidebar map

  • New task — start a fresh conversation
  • Projects — folders + ongoing work
  • Scheduled — recurring + one-off tasks
  • Live artifacts — HTML pages with fresh data
  • Dispatch — long-running background work (Beta)
  • Customize — skills, connectors, plugins

Keyboard shortcuts

  • cmd+N — new conversation in current window
  • / in input — trigger a specific skill explicitly
  • + in input — attach a file

When to use which

  • Quick question, no files → Chat mode
  • Real work touching files/tools → Cowork
  • Long task that runs while you do other things → Dispatch
  • Recurring work → Scheduled task
  • Data that changes over time → Live artifact
  • Multiple parallel tasks → Subagents (Cowork picks them)

Top 10 prompts

  • "Give me a morning briefing — calendar, email, what matters today."
  • "Prep me for tomorrow's first meeting with attendee and history context."
  • "Triage my unread Slack and email. Group by urgency."
  • "Every weekday at 7am, run the morning briefing above."
  • "Build me a live artifact showing my open [tracker] tickets."
  • "Read every PDF in this folder and summarize each in one line."
  • "Research [company] — one-page brief."
  • "Critique this design / write a PRD for X / run an SEO audit on Y" (skill triggers)
  • "Remember that I prefer [tone/format/preference]."
  • "What do you currently remember about me?"

Skill patterns to know

  • /write-spec — turn an idea into a structured PRD
  • /design-critique, /accessibility-review — design feedback
  • /review-contract, /triage-nda — legal triage
  • /account-research, /call-prep — sales prep
  • /campaign-plan, /brand-review — marketing
  • /skill-creator — build your own skill

Safety patterns

  • Before bulk operations: "show me what you'd do before doing it"
  • Before sending: "draft, don't send"
  • Before deleting: "move to a quarantine folder first, don't delete"
  • Memory check: "show me your memory files"

The 5-minute setup

1. Pick a working folder — ~/Documents/CoworkDemo is fine for tests. 2. Connect one tool you live in (Gmail, Calendar, or Slack). 3. Run a real task — don't test with a toy prompt. 4. If it worked, schedule it. 5. As friction comes up, connect more tools.
