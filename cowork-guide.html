The shape of it

Three layers, one assistant.

Your files

Pick a folder on your computer. Claude can read, write, and edit files in it — docs, spreadsheets, decks, PDFs, code, anything.

Your computer

With your permission, Claude can take screenshots, click, type, and drive native apps — Finder, Notes, Maps, third-party software, anything.

Your tools

Connect Gmail, Slack, Notion, Linear, Figma, Drive, and 30+ more. Claude reads from and acts in your real workplace.

Quick start

Get to your first win in five minutes.

If you do nothing else from this guide, do these three things. After that, Cowork can already do useful work.

1

Pick a folder for Claude to work in

Cowork needs a workspace. Pick a folder where finished outputs should land — a project folder, a Documents subfolder, anything sensible. Claude can read and write files there. Files outside that folder are off-limits unless you grant access.

Try saying
"Let's use my ~/Documents/Projects/marketing folder for this session."
Heads up: Cowork has two file areas. The folder you pick (where your real work lives) and a hidden temporary scratchpad (where Claude drafts things you don't need to see). Final outputs always land in your folder with a clickable link.
2

Connect one tool you use every day

A connected tool turns Claude from a smart chat into a coworker. Start with the one you live in. Gmail, Calendar, Slack, Notion, or Linear are common first picks.

Try saying
"Connect Gmail so you can help me triage email."
How it works: Claude opens an OAuth flow in your browser. You approve, and the connection persists across sessions. You can revoke it anytime.
3

Run one real task end to end

Don't test Cowork with a toy prompt. Ask it to do something you'd otherwise do yourself today. The point is to see it complete a real loop — read source data, do the work, hand you a deliverable.

Files & folders
Read, write, edit, and reorganize files in a folder you pick.
What it is
Claude can read every file in the folder you select (Word, Excel, PowerPoint, PDF, CSV, code, plain text, images) and create or edit files in that same folder. New deliverables show up as clickable links right in chat.
When to reach for it
Any time the work touches files you'd otherwise open by hand. Reports, spreadsheet cleanup, batch renames, format conversions, content drafts that live as documents.
Walkthrough
  1. 1
    Open Cowork and select a folder. Claude can now see what's in it.
  2. 2
    Ask a question about the contents ("what's in this folder?", "summarize the deck named Q2 launch").
  3. 3
    Ask for an output. Claude builds the file in your folder and gives you a link to open it.
  4. 4
    Iterate. "Make the executive summary tighter." "Add a chart for the regional breakdown."

A real Linux environment for Python, Node, and shell scripts.
What it is
An isolated Linux environment with Python, Node, and common CLI tools. Claude can install packages, process data, run analyses, and emit files into your workspace.
When to reach for it
Anything computational: cleaning datasets, transforming files, running statistical analysis, generating charts, scraping structured data, or one-off scripts you'd otherwise write by hand.

    Hand Claude a file (CSV, JSON, log, anything) or describe a computation.
  2. 2
    Claude writes the script and runs it. You see the output (and the code, if you ask).
  3. 3
    Results get written back into your folder — a new spreadsheet, a chart image, a cleaned dataset.

Control your computer
Screenshots, clicks, typing in native desktop apps.
What it is
Claude can take a screenshot of your screen and drive your computer with mouse and keyboard. Useful for native apps like Notes, Finder, Maps, System Settings, Photos, or any third-party desktop app without an API.
When to reach for it
When the work is in a desktop app and there's no MCP for it. Cross-app workflows (drag from one app to another). Anything where the only interface is the GUI.
Walkthrough
  1. 1
    Ask Claude to do something on your desktop. It will request permission for the apps it needs.
  2. 2
    Approve each app explicitly. Some apps get restricted tiers (browsers and IDEs are visible but read-only or click-only by default — for safety).
  3. 3
    Claude takes screenshots to look at your screen, then clicks and types. You can watch it work.
  4. 4
    You can interrupt anytime by moving the mouse or telling it to stop.
Safety guardrails: Claude will not execute trades, send money, or initiate transfers — even in apps where it has full access. For browsers it defaults to read-only (use the Chrome extension for navigation).

Drive your browser
Navigate, fill forms, extract page content via Claude in Chrome.
What it is
A Chrome extension that lets Claude see and act on web pages through the DOM, not by clicking pixels. Much faster and more reliable than computer use for web apps.
When to reach for it
Any web app that doesn't have its own MCP. Booking flights, filling expense reports, scraping a SaaS dashboard, navigating internal tools.
Walkthrough
  1. 1
    Install the Claude in Chrome extension (Claude will prompt you the first time).
  2. 2
    Ask Claude to do something in a browser. It will open or use an existing tab.
  3. 3
    It reads the page, clicks the right elements, fills inputs, and reports back what it did.

Connect your apps (MCP)
Gmail, Slack, Notion, Linear, Figma, Drive — 30+ connectors.
What it is
MCP (Model Context Protocol) connectors are official integrations with apps you already use. Each gives Claude read and write access through proper APIs — fast, structured, permission-gated.
When to reach for it
Whenever you'd otherwise alt-tab into an app to look something up, paste it somewhere, or send something. Connect it once, then ask Claude.
Walkthrough
  1. 1
    Ask Claude what connectors are available, or directly ask it to connect a specific tool.
  2. 2
    Claude opens an OAuth approval page in your browser. Confirm.
  3. 3
    That connector now persists across sessions. Claude can use it any time.
  4. 4
    To revoke, ask Claude to disconnect it (or do it from the connectors panel).

Skills
Pre-built playbooks for specific kinds of work.
What it is
Skills are recipes Claude knows how to follow. Each one packages domain expertise — how a marketer writes a campaign brief, how a lawyer triages an NDA, how an engineer writes a postmortem — and turns it into a repeatable workflow.
When to reach for it
When the output has an industry-standard shape. Claude knows what good looks like and produces structured, complete deliverables instead of generic prose.
Walkthrough
  1. 1
    You don't have to memorize skill names. Phrase the ask naturally — "write a PRD for X", "review this contract" — and Claude picks the right skill automatically.
  2. 2
    The skill kicks in and asks any clarifying questions it needs.
  3. 3
    You get a structured deliverable — PRD, brief, audit report, etc. — formatted the way that kind of doc is supposed to look.
  4. 4
    If you want to see what's installed, ask: "what skills do I have?"

Persistent memory
Remembers facts about you across sessions — preferences, projects, feedback.
What it is
A file-based memory system that survives across conversations. Claude builds up a picture of who you are, how you like to work, what projects you're in, and what feedback you've given.
When to reach for it
It runs automatically. You can also explicitly tell Claude to remember (or forget) something.

  3. 3

  4. 4
    Audit any time: "what do you remember about me?" or "show me your memory file for project X."

Worked example

Activate a role-bundle plugin.

Five steps, real screenshots. We're using the Design plugin in this walkthrough — the exact same flow works for Sales, Marketing, Engineering, Legal, Ops, Product Management, Customer Support, or Brand Voice. Pick whichever matches your role.

1

Open Customize

From any Cowork window, click Customize in the left sidebar (briefcase icon, near the bottom). You'll land on the page below.

The "Customize Claude" landing page. Three cards correspond to the three ways to extend Cowork — connectors, skills, and plugins.

2

Click "Browse plugins"

The Directory opens to the Plugins tab, showing role bundles from Anthropic & Partners. Each card lists the publisher, install count, and a one-line description of what the bundle covers.

Ten role bundles available right now. Each one packages 5–10 skills plus recommended connectors for that job function.

3

Open the plugin you want

Click any plugin card to see what's inside. The detail view shows:

  • ·
    "Try asking…" examples — sample prompts that fire each skill, so you can see what the bundle actually does.
  • ·
    Skills chips — the slash-command names you can use to trigger each skill directly.
  • ·
    Connectors chips — tools the plugin works best with. Not required, but the skills get richer when these are connected.

The Design plugin: 5 example prompts, 7 skills (/design-critique, /accessibility-review, and 5 more), and 9 recommended connectors (Slack, Figma, Linear, Asana, Atlassian, Notion, Intercom, Google Calendar, +1).

4

Hit Install

Top-right of the plugin detail view: an Install button. Click it. The plugin downloads and registers all its skills in your Cowork instance.

If you've already installed the plugin, the button reads Uninstall and a second Manage button appears next to it. Manage lets you turn individual skills on/off without uninstalling the whole bundle — useful when you want, say, 5 of the 7 Design skills but not all.

Connectors aren't auto-installed. To finish the setup, switch to the Connectors tab in the Directory and connect the tools the plugin recommends. OAuth flow per service, one minute each. Skip any you don't use.

5

Use it — two ways

Once installed, the plugin shows up in your left sidebar under Personal plugins, with every skill nested below it. Skills activate in two ways:

Natural language (default). Just describe what you need. Each skill has a description telling Cowork when to fire. "Critique this design for me — I'm worried about hierarchy" auto-activates /design-critique without you naming it.
Slash command (explicit). Type the skill's name with a leading slash to force it. /design-handoff jumps straight into the developer handoff workflow even if your prompt was ambiguous about which skill to use. Best for when you know exactly what you want.

That's it. The plugin is live. Skills will trigger on their own from now on whenever your prompts match their patterns — no further setup needed.

One more thing — making your own. The plugin marketplace is the easy path, but you can also build custom skills. From Customize → Skills, click + and pick Create with Claude (conversational builder), Write skill instructions (a form for the name, description, and instructions), or Upload a skill (drop in a .skill zip). Custom skills activate the same way as bundled ones — natural language match or slash command.
Hands-on

Now go build something.

Reading about Cowork is orientation. Building with Cowork is when it becomes useful. Eight projects below, each teaching a different aha. Pick one, ladder up.

Where to start

If you only do one: #1 Morning briefing — scheduled tasks are Cowork's productivity unlock and most people don't reach for them naturally. If you've got time for three, ladder #1 → #5 (custom skill) → #3 (live artifact). That sequence takes you from "Cowork shows up for you" to "Cowork is yours to shape" to "Cowork ships things that keep working."

01
30 min Beginner

Morning briefing on autopilot

The aha: scheduled tasks are the real productivity unlock — Cowork shows up for you instead of you remembering to ask.

Wire up a daily-running task that pulls from your calendar, email, and any open task tracker. Every morning at the time you set, Cowork delivers a 30-second read on what matters today. Once you have one, you stop opening apps to check.

02
1 hr Beginner

AI inbox triage across three apps

The aha: multi-connector synthesis collapses three apps into one prompt — "what's on fire" becomes a one-liner.

Connect Gmail, Slack, and Calendar. Practice prompts that pull across all three at once — unanswered threads, conflicting holds, overdue replies. The real skill you build: phrasing asks that demand synthesized answers, not single-source lookups.

03
1 hr Intermediate

Live project tracker artifact

The aha: live artifacts beat static reports — the data refreshes itself every time you open the page.

Pick a tracker you already use (Linear, Asana, Notion, ClickUp). Build a live artifact that shows your open items grouped however you think about them, with the data pulled fresh each time you open the page. Sharable URL. Replaces "let me check Linear" with "open the page."

04
1.5 hr Intermediate

Meeting prep system

The aha: Cowork chains tools without you orchestrating it — calendar + email + history + transcripts assemble into a brief on their own.

Pick a meeting on your calendar tomorrow. Ask Cowork to build a one-page brief: who's attending, what you've discussed before, what's outstanding, what to ask. Once the pattern works, schedule it to auto-run the evening before each weekday.

05
1 hr Intermediate

Build your first custom skill

The aha: when you turn a recurring prompt into a skill, Cowork stops being a tool and starts being your collaborator.

Pick a task you ask Cowork to do at least once a week. Capture how you want it done — tone, format, what to include, what to skip — and save it as a custom skill. Next time you describe the task, the skill activates automatically. Test, refine instructions, refine again.

06
1 hr Beginner

Document Q&A across a folder

The aha: files + skills together turn 50 PDFs into a searchable knowledge base — no upload UI, no embedding setup, no chunking.

Drop a folder of source documents (research papers, internal docs, vendor contracts, whatever). Practice asking questions across them — "find every mention of X," "summarize the three documents that discuss Y," "compare the pricing in these contracts." Useful sub-skill: telling Cowork to cite which file each answer came from.

07
2 hr Advanced

Quarterly review compiler

The aha: cross-tool synthesis at scale — Cowork pulls from git, Linear, Slack, and calendar, then turns the raw history into a polished review doc.

At the end of a quarter, ask Cowork to compile everything you shipped. Closed tickets, commits, key threads, notable meeting notes. Output a Word doc with sections for shipped work, themes, lessons learned, recommendations. The output is sharable as-is or becomes the source for your actual review write-up.

08
2 hr Advanced

Connect a custom MCP

The aha: the MCP registry is the unlock for non-mainstream tools — Cowork extends to whatever you need, not just what's pre-built.

Pick a tool you use that Cowork doesn't have an official connector for. Search the MCP registry to see if a community MCP exists. If yes, install and connect. If no, this is the project: use the mcp-builder skill to build a minimal MCP for it. Result: Cowork can now reach a system it couldn't before.

Tips & patterns

Get more out of Cowork.

Specificity

Tell it what "done" looks like.

"Summarize this" leaves Claude guessing about length, format, and audience. "Summarize this for a CFO in 5 bullets, under 100 words" gets you the right answer first try.

Iteration

Treat the first draft as a starting point.

Cowork is fast at revision. Don't try to nail it in one prompt — get a v1, then sharpen with "make it more direct", "cut the intro", "add a chart".

Context

Connect more, prompt less.

Every connector you add reduces what you need to type. Once Gmail and Calendar are connected, "prep me for tomorrow's meetings" becomes a one-liner.

Memory

Train it on your preferences.

When Claude does something well, say so. When it overdoes it, correct it. Both go into memory. Within a week it'll feel personalized.

Safety

Verify before destructive actions.

Before letting Claude bulk-rename, bulk-delete, or send emails to multiple people, ask for a dry run first: "show me what you'll do before doing it."

Delegate

Push hard work to scheduled tasks.

If you find yourself asking the same question every morning, every Monday, or every month — that's a scheduled task waiting to happen.
